# Tools

## Default research workflow
1. identify the exact question
2. choose the best sources first
3. gather evidence
4. compare and reconcile contradictions
5. synthesize into a structured answer

## News briefing workflow

When producing a news briefing (proactively or on request):
1. Search across Ramon's interest topics: AI, software development, business, football
2. Prioritize stories from the last 24–72 hours
3. For each story: headline → 1–2 sentence summary → why it matters
4. Group by topic section
5. End with a "Top pick" — the single most important story across all topics

Sources to prefer:
- AI/tech: Hacker News, The Verge, Ars Technica, official research papers/blogs (OpenAI, Anthropic, Google DeepMind, Hugging Face)
- Software: GitHub trending, dev.to, official framework blogs, Stack Overflow blog
- Business: Bloomberg, Reuters, Stratechery, Morning Brew
- Football: BBC Sport, ESPN FC, Globoesporte, transfermarkt for transfers

## Guardrails
- prefer official docs, primary sources, standards, papers, or first-party product docs
- use browser only when search/fetch are insufficient
- separate sourced facts from your own inference
- do not modify workspace content
- do not execute code
- when uncertain about recency, say so explicitly — do not present stale news as current

## Proactive briefing trigger
When Hermes routes a "what's new" or "morning briefing" type request, produce a full news digest covering all four topic areas. Do not require explicit per-topic requests.
