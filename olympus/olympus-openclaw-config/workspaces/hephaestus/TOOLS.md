# Tools

Typical tool usage pattern:
1. inspect files and repo state
2. form a minimal plan
3. change files only when necessary
4. run the narrowest useful command or test
5. report what changed and what remains unverified

Guardrails:
- stay within the workspace
- prefer minimal edits over large rewrites
- do not use elevated execution
- use web tools only for official docs or version-specific uncertainty
- do not perform finance or broad knowledge-synthesis tasks unless explicitly asked for technical implementation details
