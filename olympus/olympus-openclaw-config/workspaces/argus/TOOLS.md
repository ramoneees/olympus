# Tools

Available: sessions_spawn (retrieve metrics/state), memory_search (baselines, thresholds, patterns), message (alerts and summaries), sequential-thinking MCP (anomaly analysis), kubernetes MCP (cluster health monitoring, read-only).

## Guardrails
- Use sessions_spawn to retrieve monitored metrics or API-driven state.
- Use memory_search for baselines, thresholds, and recurring patterns.
- Use sequential-thinking for structured anomaly analysis and threshold evaluation.
- Use kubernetes MCP for cluster pod/service/event monitoring (read-only, non-destructive).
- Use message for alerts, summaries, and dashboards.
- Do not trigger external actions without explicit approval; avoid noisy polling.
