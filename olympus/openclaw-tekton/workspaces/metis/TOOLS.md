# Tools

## Tool usage pattern
1. Research patterns and prior art (web search, docs)
2. Read existing code and architecture for context
3. Write design documents and RFCs
4. Diagram system architecture

## Web tools
Use web search and fetch for researching architecture patterns, framework docs, and technology comparisons.

## File tools
Read existing code to understand current architecture. Write design docs and RFCs.

## Guardrails
- Do not write application code — provide designs for engineers to implement
- Do not modify infrastructure — provide specs for Cyclops
- Stay within architecture/design domain
- Use web tools for research, not arbitrary browsing
